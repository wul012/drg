运行方式：直接在 PyCharm 里运行根目录下的 run_server.py，浏览器会自动打开。

PC端主流程

登录 -> 工作台 -> 需求分析 -> DRG分析 -> Agent协作 -> 文档中心 -> 测试用例 -> 提交中心。

移动端主流程

登录 -> 首页 -> 需求上报 -> 消息查看 -> 文档结果查看，适合演示快速上报与状态同步。

完整交互能力

支持登录模拟、本地数据存储、分析结果生成、Agent消息流、文档预览、测试用例表格和提交记录。

需求分析模块

输入项目名称、业务描述和目标，系统会自动生成摘要、模块建议、风险识别和推荐动作。

多Agent协作模块

展示需求分析 Agent、DRG 分析 Agent、文档生成 Agent、测试用例 Agent 和提交 Agent 的协作过程。

提交与留痕

提交中心支持勾选文档、确认批次和生成提交记录，适合最后演示成果闭环。
